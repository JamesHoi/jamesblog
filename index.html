Hexo-admin添加云图片

在用hexo-admin后台写markdown的时候正想要加图片的时候，发现操作十分麻烦，需要用SFTP软件上传到指定位置，经过一轮的资料搜索，找到了几个方法。
1.hexo-admin-ehc
2.hexo-admin-qiniu

但最后发现，ehc搞不定，qiniu需要实名注册。（其实是hexo-admin可以的，详见方法三）
依照这个思路，找到了另外两个解决方法：
1.使用cloudinary保存图片
2.建立github项目用来保存图片
接下来本文会详细介绍这两个方法

方法一.使用cloudinary保存图片

  1. 进入cloudinary，点右上角注册

  2. 注册后进入，初始化设置（当然可以选择跳过）

  3. 建立hexo文件夹
    点击左上角这个图标

    点击新增文件夹，命名为hexo （实际上可以跳过这一步，不过有文件夹好管理一些）

    然后拖拉图片上传
    图片上传后，点击图片的左上角复制链接

方法二.建立github项目用来保存图片

既然cloudinary这类都可以，那github当然也不例外

  1. 建立github项目
    用浏览器打开github项目,点击需要用的图片（怎么用github请自行百度）
  2. 获取图片地址
    图片地址格式：https://raw.githubusercontent.com/用户名/项目名称/你的分支(默认master)/图片路径/图片名称

方法三.直接复制图片到hexo-admin

我发现原来我搞了这么多其实hexo-admin还是可以添加图片的，只是没有按钮

  1. 右键复制电脑中的图片
  2. Ctrl+V复制在hexo-admin写markdown的位置

    图片会自动上传到 hexo项目路径/source/images/
    更多设置详见 hexo-admin后台-Settings

真正的完美解决centos7.5的图形界面中文乱码的方法

网上看了很多很多的办法，就是搞不定乱码的情况。本文目的是希望后人能够迅速解决这个问题，此方法在Centos 7.5亲测有效。

一键解决：https://github.com/JamesHoi/pane-fixer-centos （下面的都可以忽略了）

参考 https://www.linuxidc.com/Linux/2013-01/77362.htm , 安装语言包
yum -y install fonts-chinese
yum -y install fonts-ISO8859-2
但是却显示错误No package fonts-chinese available.，找不到安装包

经过一番寻找，找到了相应需要的fonts-chinese-3.02-12.el5.noarch.rpm和fonts-ISO8859-2-75dpi-1.0-17.1.noarch.rpm（请前往下载），显示错误缺乏依赖包chkfontpath
查询chkfontpath官网，下载wget http://ftp.altlinux.org/pub/distributions/ALTLinux/Sisyphus/x86_64/RPMS.classic//chkfontpath-2.0.3-alt1.x86_64.rpm

最后安装
yum install chkfontpath-2.0.3-alt1.x86_64.rpm
yum install fonts-ISO8859-2-75dpi-1.0-17.1.noarch.rpm
yum install fonts-chinese-3.02-12.el5.noarch.rpm

原文网址: https://blog.csdn.net/baidu_38635190/article/details/88922732
